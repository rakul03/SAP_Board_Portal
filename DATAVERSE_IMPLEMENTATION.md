## Dataverse Tables & Fields

### 1. Portfolio Owners (`sap_portfolioowner_saps`)
```
sap_portfolioowner_sapid    GUID (Primary Key)
sap_owner_id                Text (Custom) - Sequential: OWNID-1001, OWNID-1002, ...
sap_ownername               Text (Required)
sap_email                   Email (Custom)
statecode                   Choice (Active/Inactive) - Default: 0 (Active)
statuscode                  Choice (Active/Inactive) - Default: 1 (Active)
```

**Key Behavior:**
- `sap_owner_id` is NOT auto-generated by Dataverse
- Must be generated by the app during record creation
- Must be stored (patched) into the record immediately after creation
- Ensures uniqueness and sequential incrementation

### 2. Initiatives (`sap_initiative_saps`)
```
sap_initiative_sapid        GUID (Primary Key)
sap_initiativename          Text (Required)
sap_category                Choice (Enum)
sap_description             Multiline Text
sap_Owner_Name              Lookup → Portfolio Owners (1:N relationship)
sap_budgetaed               Currency (AED)
sap_implementer             Text
sap_status                  Choice (Active/Pending/Delayed/Completed)
sap_urgency                 Choice (Low/Medium/High)
sap_currentprocessasis      Multiline Text
sap_enhancedprocesstobe     Multiline Text
sap_comments                Multiline Text
sap_demandnumber            Text
statecode                   Choice (0=Active, 1=Inactive)
statuscode                  Choice (1=Active, 2=Inactive)
```

**Category Options:**
- 100000000: "AIs"
- 100000001: "Enhancements"
- 100000002: "Projects"
- 100000003: "Licenses"
- 100000004: "Services"
- 100000005: "Securities"
- 100000006: "ProductReplacements" (note: no space in Dataverse enum)
- 100000007: "Infrastructure"
- 100000008: "Others"

### 3. Audit Logs (`sap_auditlog_saps`)
```
sap_auditlog_sapid          GUID (Primary Key)
sap_initiativename          Lookup → Initiatives (1:N relationship)
sap_log_date                DateTime
sap_log_description         Multiline Text
sap_log_severity            Choice (High/Low/Medium)
sap_eventname               Text (Currently used for owner tracking)
statecode                   Choice (0=Active, 1=Inactive)
statuscode                  Choice (1=Active, 2=Inactive)
```

**⚠️ Schema Issue:** Per the specification, audit logs should have:
- `sap_owner_id` (Text) - Missing, needs to be added
- `sap_ownername` (Text) - Missing, needs to be added  
- `sap_owneremail` (Email) - Missing, needs to be added

Currently, owner information is temporarily stored in `sap_eventname`.

## DataverseService API

### Owner Operations

```typescript
// Initialize the owner ID counter (call once on app start)
await dataverseService.initializeOwnerIdCounter()

// Get all owners
const owners: Owner[] = await dataverseService.getOwners()

// Create owner with auto-generated ID
const owner = await dataverseService.createOwner(name, email?)

// Delete owner
await dataverseService.deleteOwner(ownerId)
```

### Initiative Operations

```typescript
// Get all initiatives
const initiatives: Initiative[] = await dataverseService.getInitiatives()

// Get single initiative
const initiative = await dataverseService.getInitiative(id)

// Create initiative
const initiative = await dataverseService.createInitiative({
  name, category, description, owner, budget,
  status, urgency, currentProcess, enhancedProcess, comments, ...
})

// Update initiative (partial fields)
await dataverseService.updateInitiative(id, { name, status, ... })

// Delete initiative (also deletes related audit logs)
await dataverseService.deleteInitiative(id)
```

### Audit Log Operations

```typescript
// Get all audit logs
const logs: AuditLog[] = await dataverseService.getAuditLogs()

// Get logs for specific initiative
const logs = await dataverseService.getAuditLogsByInitiativeId(initiativeId)

// Create audit log
const log = await dataverseService.createAuditLog({
  initiativeId, initiativeName, logDate, logDescription,
  logSeverity, ownerName, ...
})

// Delete audit log
await dataverseService.deleteAuditLog(id)
```

## Owner ID Generation Strategy

The `DataverseService` maintains an owner ID counter:

1. **Initialization:** On first call, fetches all existing owners and finds the highest ID
2. **Sequencing:** Increments from the highest number (e.g., 1000 → 1001 → 1002)
3. **Format:** Always returns `OWNID-{number}` (e.g., OWNID-1001)
4. **Storage:** The generated ID is stored in the `sap_owner_id` field during creation
5. **Uniqueness:** Dataverse constraints should prevent duplicate IDs

## Error Handling

The DataverseService includes:
- Try/catch blocks for all operations
- Detailed error logging to console
- Error throwing (not silent failures)
- Context for debugging

```typescript
try {
  const owner = await dataverseService.createOwner(name)
} catch (error) {
  console.error('Failed to create owner:', error)
  // Handle error UI feedback
}
```

## Mapping Between Models

The service handles conversion between:
- **App types** (Initiative, AuditLog, Owner) - Used by React components
- **Dataverse types** (Sap_initiative_saps, Sap_auditlog_saps, etc.) - Used by API

### Category Mapping Example
```
App Type: "Enhancements"
  ↕
Dataverse Code: 100000001
```

### Status Mapping Example
```
App Type: "Active"
  ↕
Dataverse Code: 100000000
```

All mappings are bidirectional to handle read/write operations.

## Common Issues & Solutions

### Issue: Owner IDs not sequential
**Solution:** Ensure `initializeOwnerIdCounter()` is called before first owner creation

### Issue: Audit log owner info not captured
**Solution:** Schema update needed - add `sap_owner_id`, `sap_ownername`, `sap_owneremail` fields

### Issue: Relationships not working
**Solution:** Verify lookup field names match: `_sap_owner_name_value` for initiatives, `_sap_initiativename_value` for audit logs

### Issue: 204 No Content responses
**Solution:** Already handled with retry logic in DataverseService
