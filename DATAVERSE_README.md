## 🔑 Key Features

### Owner ID Generation
- Automatic sequential generation: OWNID-1001, OWNID-1002, ...
- Initialized from existing records on app startup
- No manual ID management needed
- Guaranteed uniqueness

### Connection Management
- Automatic health checks every 5 minutes
- Real-time status monitoring in components
- Graceful error handling with retry logic
- Connection status visible to entire app

### Error Handling
- Automatic retry with exponential backoff
- User-friendly error messages
- Detailed logging for debugging
- Recoverable vs. permanent error detection

### Data Persistence
- All operations sync to Dataverse
- No localStorage dependency
- Data persists across page refreshes
- Automatic audit logging available

## 📊 API Quick Reference

### DataverseService (CRUD Operations)

```typescript
import { dataverseService } from '../services/DataverseService';

// Owners
await dataverseService.getOwners()           // Owner[]
await dataverseService.createOwner(name, email?)  // Owner
await dataverseService.deleteOwner(id)       // void

// Initiatives
await dataverseService.getInitiatives()      // Initiative[]
await dataverseService.getInitiative(id)     // Initiative
await dataverseService.createInitiative(data)// Initiative
await dataverseService.updateInitiative(id, data)  // void
await dataverseService.deleteInitiative(id)  // void

// Audit Logs
await dataverseService.getAuditLogs()        // AuditLog[]
await dataverseService.getAuditLogsByInitiativeId(id)  // AuditLog[]
await dataverseService.createAuditLog(log)   // AuditLog
await dataverseService.deleteAuditLog(id)    // void
```

### DataverseConnection (Lifecycle)

```typescript
import { dataverseConnection } from '../services/DataverseConnection';

await dataverseConnection.initialize()       // Initialize on app start
dataverseConnection.isConnected()            // boolean
await dataverseConnection.checkConnection()  // boolean
dataverseConnection.getStatus()              // ConnectionStatus
dataverseConnection.dispose()                // Cleanup
```

### React Hooks (Component Level)

```typescript
import { useInitializeDataverse, useDataverseConnection } 
  from '../hooks/useDataverseConnection';

// In App.tsx
const { initialized, initError } = useInitializeDataverse();

// In any component
const { isConnected, lastChecked, forceCheck, error } 
  = useDataverseConnection();
```

### Error Handler (Advanced)

```typescript
import { DataverseErrorHandler, withDataverseErrorHandling }
  from '../services/DataverseErrorHandler';

// Retry with backoff
await DataverseErrorHandler.retry(operation, maxRetries);

// Parse error
const error = DataverseErrorHandler.parseError(caught);

// Get user message
const msg = DataverseErrorHandler.getUserMessage(error);

// Wrapped operation
await withDataverseErrorHandling(operation, 'context');
```

## 🆘 Common Issues

| Problem | Solution |
|---------|----------|
| "Dataverse not connecting" | Check internet, verify Power Platform environment |
| "Owner ID not generating" | Verify `initializeOwnerIdCounter()` called |
| "Data not saving" | Ensure DataContext awaits DataverseService calls |
| "Type errors in DataContext" | Check all service methods return correct types |
| "Errors not showing to user" | Use DataverseErrorHandler.getUserMessage() |
